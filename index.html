1. O QUE O SCRIPT FAZ (ANÁLISE TÉCNICA COMPLETA)

ATENÇÃO: Este é um payload malicioso de XSS avançado. Uso apenas em ambientes autorizados.

Arquitetura do Payload

O phantom_zero.min.js é um IIFE (Immediately Invoked Function Expression) que roda no contexto da vítima assim que injetado.

Etapas de Execução

  1. Inicialização Invisível: Remove console.log, define window.phantom = null
  2. Anti-Análise: Detecta VM, Selenium, Headless via isSandbox()
  3. DOM Shadow: Clona o DOM real para um Shadow DOM fechado
  4. WASM Obfuscation: Carrega módulo WebAssembly para mutação em tempo real
  5. Persistência: Armazena payload criptografado em 5 locais
  6. Coleta de Dados: Captura cookies, forms, fingerprint, WebRTC, bateria
  7. Exfiltração: Envia dados via 5 canais simultâneos
  8. C2: Conecta via WebSocket com fallback DNS
  9. AI Mutation: Muta o próprio código a cada 60s
  10. Self-Destruct: Apaga tudo após 1h

Passo 2: Baixar o Framework

git clone https://github.com/ghostintheshell/phantom-zero.git
cd phantom-zero

Passo 3: Configurar DNS Local

sudo bash -c 'cat > /etc/hosts.d/phantom.conf' << EOF
127.0.0.1 c2d.ghost.local
127.0.0.1 tunnel.phantom.local
127.0.0.1 backup.c2.ghost.local
127.0.0.1 c2.mirror.local
EOF

# Ative
sudo systemctl restart dnsmasq

Passo 4: Iniciar o C2 Server

cd c2-server
npm install
node server.js &
